# Точки распространения (CDP / AIA / SIA)

Сертификаты содержат несколько расширений, указывающих, где скачать
связанные с ними артефакты.

## CRL Distribution Points (CDP)

Один или несколько URL, по которым публикуется CRL — список
отозванных сертификатов, выпущенных тем же УЦ. При path validation
клиент скачивает CRL по этому адресу и проверяет, нет ли серийного
номера сертификата в списке.

В OwnCA URL задаётся:

* На уровне УЦ (поле `crl_url`, см. [Управление УЦ](cas.md)).
* На уровне [профиля сертификата](cert_profiles.md) — переопределяет
  УЦ-уровень для конкретных категорий сертификатов.

> Отсутствие CDP допустимо для УЦ, у которых отзыв проверяется только
> через OCSP, но большинство клиентов TLS требуют наличие CDP.

## Authority Information Access (AIA)

Указывает два способа доступа:

* **`caIssuers`** — где скачать сертификат выпустившего УЦ (для
  построения цепочки).
* **OCSP** — адрес OCSP-респондера.

В OwnCA — поля `aia_url` и `ocsp_url` на УЦ.

## Subject Information Access (SIA)

Применяется обычно только в сертификатах **УЦ**: указывает, где
находится репозиторий выпущенных им сертификатов.

В OwnCA — поле `sia_url` на [Управлении УЦ](cas.md).

## Freshest CRL (Delta CRL Distribution Point)

Содержит URL **delta-CRL** — частичного списка отзывов между полными
публикациями. Поддерживается полем `freshest_crl_url`.

## Issuer Alternative Names

Дополнительные имена УЦ в той же форме, что и SAN (DNS, URI, e-mail и
т. д.). В OwnCA — поле `issuer_alt_names` на УЦ.

## Резолюция значений в OwnCA

При выпуске сертификата применяется правило приоритета:

1. **Профиль сертификата** (если содержит непустое значение для поля).
2. **УЦ** (значения из карточки УЦ).
3. Пустое значение → расширение не добавляется.
