# Профили сертификатов

Страница `/cert-profiles/` — шаблоны расширений сертификатов. Профиль
определяет, какие Key Usage / Extended Key Usage и пользовательские
OID-поля будут включены в выпускаемый сертификат.

Профили **глобальные** — они не привязаны к конкретному УЦ. Любой
профиль можно использовать с любым УЦ на форме выпуска.

## Список профилей

| Колонка | Описание |
|---|---|
| Name | Внутренний идентификатор (slug, `[a-z0-9_]+`). |
| Display name | Человекочитаемое имя. |
| Description | Произвольный текст. |
| Key Usage | Краткая сводка KU (например, `dS, kE`). |
| Extended Key Usage | Строка EKU. |
| CA | Метка `CA`, если профиль является CA-профилем (basicConstraints CA:TRUE). |
| In use | `<сертификатов> / <ожидающих заявок>`. |

## Структура профиля

### Key Usage

Битовая маска, описывающая, какие операции допустимы для ключа:

* `digitalSignature` — подпись данных (handshake, JWT, и т. д.).
* `nonRepudiation` (`contentCommitment`) — юридическая значимость.
* `keyEncipherment` — шифрование симметричного ключа.
* `dataEncipherment` — прямое шифрование данных.
* `keyAgreement` — выработка общего ключа (DH/ECDH/VKO).
* `keyCertSign` — подпись сертификатов (только для УЦ).
* `cRLSign` — подпись CRL.
* `encipherOnly` / `decipherOnly` — уточнения для `keyAgreement`.

Подробнее — [Key Usage и Extended Key Usage](key_usage.md).

### Extended Key Usage

Список OID-ов с прикладным назначением. Часто используемые:

| OID | Назначение |
|---|---|
| `1.3.6.1.5.5.7.3.1` | TLS Server Authentication |
| `1.3.6.1.5.5.7.3.2` | TLS Client Authentication |
| `1.3.6.1.5.5.7.3.3` | Code Signing |
| `1.3.6.1.5.5.7.3.4` | Email Protection (S/MIME) |
| `1.3.6.1.5.5.7.3.8` | Time Stamping |
| `1.3.6.1.5.5.7.3.9` | OCSP Signing |

### Пользовательские OID-поля

Профиль можно связать с записями реестра Custom OID. Каждое поле
описывает один атрибут subject DN, SAN-элемент или произвольное
расширение, заполняемое при выпуске.

### Дополнительные параметры

| Поле | Описание |
|---|---|
| Is CA profile | Если включено, профиль добавляет `basicConstraints = CA:TRUE` — для сертификатов подчинённого УЦ. |
| Extra extensions | Строки расширений, добавляемые в секцию расширений сертификата дословно. |

## Кнопка «Copy»

На странице редактирования профиля рядом с **Delete** находится кнопка
**Copy**. Она создаёт полную копию текущего профиля (вместе со связями
к OID-полям) под уникальным именем и перенаправляет на редактирование
новой копии — удобный шаблон для быстрого создания варианта
существующего профиля.
