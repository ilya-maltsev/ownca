# Разбор CSR

Парсер CSR — внутренний помощник формы [Выпуск произвольного
сертификата](custom_cert_issue.md). Отдельной страницы «CSR parse»
нет; endpoint `/certificates/csr-parse/` — авторизованный AJAX-
вызов, принимающий PEM CSR и возвращающий разобранные поля в JSON.
Форма Cert Issue отправляет туда CSR и использует ответ для
автозаполнения полей subject / SAN и показа сырого текста.

## Что извлекает парсер

В ответе возвращаются следующие ключи (пустая строка / пустой
список, если соответствующий атрибут отсутствует):

| Поле | Описание |
|---|---|
| `common_name` | Атрибут CN из subject DN. |
| `country` / `state` / `locality` / `organization` / `unit` | Прочие атрибуты subject DN. |
| `public_key_algorithm` | Например, `rsaEncryption`, `id-ecPublicKey`, `GOST R 34.10-2012`. |
| `signature_algorithm` | Алгоритм подписи самого CSR. |
| `paramset` / `paramset_raw` | Идентификатор parameter set ГОСТ (`A` / `B` / `C` / `XA` / `XB`) и сырое значение, как его вывел openssl. |
| `san_dns` | DNS-записи из Subject Alternative Name. |
| `san_ip` | IP-записи из Subject Alternative Name. |
| `requested_ku` | Биты Key Usage, запрошенные в CSR (только для справки). |
| `requested_eku` | OID-ы Extended Key Usage, запрошенные в CSR (только для справки). |
| `raw_text` | Полный вывод `openssl req -text -noout` для отображения. |

Прочие типы SAN (email, URI, otherName) этот помощник не выделяет.
Если они нужны в выпускаемом сертификате, добавьте их вручную в
соответствующие поля формы Custom Cert Issue.

Вход должен быть в PEM. DER не поддерживается.

> **Примечание по безопасности.** Запрошенные в CSR расширения
> всегда отбрасываются при подписании. Результат парсера носит
> информационный характер — он показывает, что CSR *запросил*, а не
> то, что попадёт в выпущенный сертификат. Расширения определяются
> выбранным профилем и свободными полями формы выпуска.
