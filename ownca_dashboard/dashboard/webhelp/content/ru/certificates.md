# Реестр сертификатов

`/certificates/` — таблица всех выпущенных сертификатов и точка входа
к операциям над конкретным сертификатом (просмотр, отзыв, продление,
удаление, скачивание).

## Фильтры

В шапке страницы — два выпадающих списка:

* **CA** — ограничить выборку сертификатами одного УЦ.
* **Status** — `active`, `revoked` или `expired`.

Поиска по колонкам и общего поля поиска нет.

## Колонки

| Колонка | Описание |
|---|---|
| Common Name | CN из subject — ссылка на страницу деталей сертификата. |
| Issuer CA | УЦ-эмитент — ссылка на страницу УЦ. |
| Profile | Снимок отображаемого имени [профиля](cert_profiles.md), сделанный в момент выпуска. |
| Algorithm | Алгоритм ключа. |
| Serial | Серийный номер (hex). |
| Valid Until | `notAfter`. |
| Status | `Active` / `Revoked` / `Expired`. |

Действия (отзыв/продление/удаление/скачивание) выполняются со
страницы деталей — переход по ссылке в колонке Common Name.

## Страница деталей

`/certificates/<uuid>/` содержит:

* Панель кнопок: **Download cert**, **Download key** (когда ключ
  хранится на сервере), **Download CSR**, **Download PEM bundle**,
  **Export PKCS#12**, **Renew**, **Revoke**, **Delete**.
* Блок деталей: subject, статус, УЦ-эмитент, профиль, алгоритм,
  серийный номер, SHA-256-отпечаток, срок действия, SAN DNS/IP,
  значения custom-OID, наличие закрытого ключа, данные об отзыве,
  ссылка на исходный сертификат (если это продление).
* Блок **Renewals issued from this certificate** — список продлений
  (когда применимо).
* Полный вывод `openssl x509 -text` в панели **X.509 details**.

Отдельной панели «История действий» и графической схемы цепочки на
странице нет.

## Отзыв

Кнопка **Revoke** разворачивает встроенную форму (без модального
окна) с селектором причины:

| Причина | Когда используется |
|---|---|
| `unspecified` | Причина не указана. |
| `keyCompromise` | Закрытый ключ скомпрометирован. |
| `CACompromise` | Скомпрометирован ключ УЦ. |
| `affiliationChanged` | Изменилась организация владельца. |
| `superseded` | Заменён новым сертификатом. |
| `cessationOfOperation` | Прекращена эксплуатация. |
| `certificateHold` | Временная приостановка. |

После подтверждения сертификат помечается отозванным, генерируется
новый CRL и создаётся запись в журнале выпусков CRL.

## Продление

Кнопка **Renew** разворачивает встроенную форму с полем **Days**.
Новый сертификат подписывается тем же УЦ с использованием исходного
CSR (и ключа, если он хранится), исходного профиля и значений OID;
меняются только даты валидности и серийный номер. Прежний
сертификат остаётся активным — отзовите его явно, если он больше не
должен считаться доверенным.

Файл исходного CSR должен по-прежнему присутствовать на диске,
иначе продление завершится ошибкой.

## Удаление

Кнопка **Delete** удаляет запись из БД и файлы сертификата / ключа /
CSR с диска. **Ограничений по статусу нет** — активные сертификаты
тоже можно удалить. Если нужна запись в CRL, используйте отзыв;
удаление просто убирает запись из реестра.
