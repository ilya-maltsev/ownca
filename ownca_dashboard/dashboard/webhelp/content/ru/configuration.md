# Конфигурация

`/system/configuration/` состоит из двух частей:

* **Режимы выпуска (Issuance modes)** — набор переключателей,
  хранящихся в базе данных, которые задают, какие семейства ключей,
  способы генерации ключа, форматы экспорта и правила профилей
  разрешены во всём приложении. Меняются здесь же, кнопкой
  **Save settings**.
* **Значения по умолчанию** и **Переменные окружения** — разделы
  «только для чтения», показывающие значения, загруженные из переменных
  окружения при старте, чтобы проверить параметры контейнера, не заходя
  в shell.

## Режимы выпуска

Эти переключатели применяются везде, где может произойти
соответствующее действие: интерфейс скрывает или фильтрует
запрещённые варианты, **и** сервер отклоняет запросы, пытающиеся
обойти интерфейс. Значения по умолчанию ниже действуют при первом
запуске; изменить их можно в любой момент кнопкой **Save settings**.

| Переключатель | По умолчанию | Что значит положение **«выкл»** |
|---|---|---|
| **Allow server-side key generation** | вкл | Приложение не генерирует закрытый ключ за вас — в форме выпуска принимается только загруженный CSR. См. [Выпуск сертификата](custom_cert_issue.md). |
| **Allow GOST keys** | вкл | ГОСТ Р 34.10-2012 (256 / 512) убирается из списков алгоритмов для новых УЦ и сертификатов. (Для работы также требуется `OWNCA_GOST_ENGINE=on`.) См. [ГОСТ-алгоритмы](gost_algorithms.md). |
| **Allow RSA keys** | вкл | RSA (2048 / 4096) убирается из списков алгоритмов. |
| **Allow ECDSA / Ed25519** | **выкл** | Когда включено — предлагаются ECDSA (P-256, P-384) и Ed25519. По умолчанию выключено, поэтому «из коробки» эти семейства нельзя выбрать для нового УЦ или сертификата. |
| **Offer `.gost.p12` export** | вкл | Скачивание в TK-26-совместимом формате `.gost.p12` (RFC 9337 + RFC 9548) скрывается на странице сертификата и отклоняется сервером. Не действует, если ключ не ГОСТ. См. [Экспорт PKCS#12](pkcs12_export.md). |
| **Allow free-form certificate issuance** | **выкл** | В форме выпуска **обязателен профиль** — вариант «Free-form (no profile)» убирается, а сервер отклоняет запросы без профиля. Когда **включено** — разрешён свободный выпуск, и ограничения профиля (KU / EKU / расширения / OID) можно обойти. См. [Выпуск сертификата](custom_cert_issue.md). |

> **Где применяются переключатели семейств ключей.** Три переключателя
> семейств (GOST / RSA / ECDSA) действуют **и** при создании УЦ
> (Authorities), **и** при выпуске конечных сертификатов (Cert Issue):
> выпадающие списки алгоритмов фильтруются по разрешённым семействам, а
> сервер отклоняет запрещённое семейство, даже если его подставить в
> запрос принудительно. **Продление** сертификата при этом **не**
> ограничивается — оно переиспользует существующий ключ, поэтому
> сертификат, выпущенный до отключения семейства, можно продлить.

## Панель «Значения по умолчанию»

Раздел «только для чтения», показывает значения по умолчанию,
загруженные из переменных окружения. Здесь ничего не редактируется;
панель отображает уже действующие значения.

| Поле | Переменная окружения | Назначение |
|---|---|---|
| Storage directory | `OWNCA_STORAGE_DIR` | Каталог, где хранятся ключи и сертификаты УЦ. По умолчанию `/var/lib/ownca`. |
| Default key algorithm | `OWNCA_DEFAULT_KEY_ALG` | Алгоритм, выбираемый по умолчанию в формах создания УЦ и сертификатов. См. [ГОСТ-алгоритмы](gost_algorithms.md). |
| Default CA validity (days) | `OWNCA_DEFAULT_CA_DAYS` | Срок жизни, проставляемый в поле **Validity (days)** формы создания УЦ (по умолчанию `3650`). |
| Default cert validity (days) | `OWNCA_DEFAULT_CERT_DAYS` | Срок жизни, используемый при продлении сертификата, если поле оставлено пустым (по умолчанию `365`). |
| CRL distribution URL | `OWNCA_CRL_DISTRIBUTION` | Информационный базовый URL — он **не** подставляется автоматически при создании УЦ; CDP по-прежнему нужно вписать вручную в форме УЦ. |

## Справочник переменных окружения

Вторая панель страницы перечисляет переменные окружения, влияющие на
приложение, в том числе те, которых нет в карточках выше:

| Переменная | Назначение |
|---|---|
| `OWNCA_STORAGE_DIR` | Корневой каталог для файлов УЦ и сертификатов. |
| `OWNCA_OPENSSL_BIN` | Путь к бинарнику `openssl` (gost-engine подгружается через `OPENSSL_CONF`). |
| `OWNCA_DEFAULT_KEY_ALG` | Алгоритм ключа по умолчанию. |
| `OWNCA_DEFAULT_CA_DAYS` | Срок действия УЦ по умолчанию. |
| `OWNCA_DEFAULT_CERT_DAYS` | Срок действия конечных сертификатов по умолчанию. |
| `OWNCA_CRL_DISTRIBUTION` | Публичный URL, по которому будут доступны CRL. |
| `OPENSSL_CONF` | Путь к `openssl.cnf`, загружающему gost-engine. |

## Применение изменений

**Режимы выпуска** сохраняются в базу данных в момент нажатия
**Save settings** и действуют со следующего запроса — перезапуск не
нужен.

**Значения по умолчанию** и **переменные окружения** устроены иначе:
они читаются из переменных окружения при старте. Чтобы их изменить,
отредактируйте `dev_env/.env` (или соответствующий файл из `demo/`) и
перезапустите контейнер — на следующем запросе страница покажет новые
значения.
