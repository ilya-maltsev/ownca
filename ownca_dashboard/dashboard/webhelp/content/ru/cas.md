# Управление УЦ

`/cas/` — корневые и подчинённые удостоверяющие центры. На странице
совмещены встроенная форма **Create new authority** и список УЦ.

## Список УЦ

| Колонка | Описание |
|---|---|
| Name | Внутреннее имя (используется в URL и интерфейсе). |
| Type | `Root CA` / `Intermediate CA`. |
| Parent | Родительский УЦ (для intermediate). |
| Algorithm | Алгоритм ключа. |
| Issued | Сколько сертификатов выпущено. |
| Valid Until | `notAfter` сертификата УЦ. |
| Status | `Active` (когда `is_enabled`) или `Disabled`. |

## Создание УЦ

Форма **Create new authority** размещена над таблицей — отдельной
кнопки «открыть форму» нет, она показана всегда. Два сценария:

1. **Root CA** — создаётся самоподписанный сертификат.
2. **Intermediate CA** — выберите **Type = Intermediate CA**, после
   чего появится поле **Parent CA**; OwnCA генерирует CSR,
   подписывает его родителем и сохраняет цепочку.

### Поля формы

| Поле | Назначение |
|---|---|
| Name * | Внутренний идентификатор (URL и UI). |
| Common Name * | CN из subject DN УЦ. |
| Type | `Root CA` или `Intermediate CA`. |
| Parent CA | Обязательно при Type = Intermediate. |
| Algorithm | `gost2012_256`, `gost2012_512`, `rsa:2048`, `rsa:4096`, `ec:P-256`, `ec:P-384`, `ed25519`. Алгоритм УЦ задаёт его **семейство ключа** (`gost`, `rsa`, `ec`, `ed25519`) и тем самым определяет, какие листовые сертификаты он сможет подписывать — см. [Custom Cert Issue](custom_cert_issue.md). |
| Validity (days) | Срок действия УЦ; по умолчанию `OWNCA_DEFAULT_CA_DAYS`. |
| Path length | Максимальная глубина цепочки под этим УЦ (пусто = без ограничения). |
| CRL Distribution Point URL | CDP, встраиваемый в выпускаемые сертификаты. |
| AIA caIssuers URL | Где скачать сертификат данного УЦ. |
| OCSP responder URL | OCSP-адрес. |
| SIA caRepository URL | Subject Information Access (актуально для sub-CA). |
| Freshest CRL URL | Адрес delta-CRL. |
| issuerAltName entries | По одной openssl-записи в строке (например, `email:ca@example.org`). |
| Country / State / Locality / Organization / Unit | Дополнительные атрибуты subject DN. |

Правила приоритета между значениями УЦ и переопределениями профиля —
см. [Точки распространения](distribution_points.md).

## Страница УЦ

`/cas/<uuid>/` содержит:

* Панель кнопок: **Download CA cert**, **Download CRL**,
  **Regenerate CRL**, **Delete**.
* Панель **Authority details** — все поля УЦ, включая текущий номер
  CRL, все URL точек распространения и счётчик выпущенных
  сертификатов с разбивкой active / revoked.
* Панель **Distribution points** — форма для редактирования
  `crl_url`, `aia_url`, `ocsp_url`, `sia_url`, `freshest_crl_url` и
  `issuerAltName` уже после создания УЦ. Изменения применяются к
  сертификатам, выпускаемым с этого момента.
* Полный вывод `openssl x509 -text` для сертификата УЦ.

### Кнопки на странице УЦ

* **Download CA cert** — PEM-файл сертификата УЦ.
* **Regenerate CRL** — создаёт новый CRL и добавляет запись в
  журнал выпусков CRL.
* **Download CRL** — отдаёт последний CRL (генерирует новый, если
  ещё не было ни одного).
* **Delete** — срабатывает только когда у УЦ **нет дочерних УЦ и
  нет ни одного выпущенного сертификата**. Отозванные и истёкшие
  сертификаты тоже считаются — их нужно удалить заранее.

## CRL

CRL автоматически перегенерируется при отзыве любого сертификата
этого УЦ, а также по нажатию **Regenerate CRL**. Каждая генерация
фиксируется записью `CrlExport`. Подробно — в разделе [Отзыв и
CRL](cert_revoke.md).

## Безопасность ключей

Закрытые ключи УЦ хранятся в каталоге `OWNCA_STORAGE_DIR` (по
умолчанию `/var/lib/ownca/`). Резервное копирование этого каталога —
обязательно.
