# Основные понятия PKI

В этом разделе кратко описаны термины, на которых строится работа
OwnCA.

## Сертификат X.509

Цифровой документ, связывающий открытый ключ с идентификатором
владельца (subject DN). Подписан удостоверяющим центром (issuer).
Основные поля:

* Серийный номер — уникальный в рамках УЦ.
* Subject / Issuer — Distinguished Name владельца и УЦ.
* Validity — `notBefore` / `notAfter`.
* Открытый ключ — связанный ключ и его алгоритм.
* Расширения — Key Usage, SAN, точки распространения и т. п.

## Удостоверяющий центр (CA)

Сущность, выпускающая сертификаты. Различают:

* **Корневой УЦ (Root CA)** — самоподписанный, корень доверия.
* **Подчинённый УЦ (Intermediate CA)** — выпущен другим УЦ; пригоден
  для выдачи конечных сертификатов и снимает нагрузку с корневого
  ключа.

## Заявка на сертификат (CSR / PKCS#10)

Контейнер с открытым ключом и атрибутами subject DN, подписанный
закрытым ключом заявителя. OwnCA принимает CSR **только в формате
PEM** — форма выпуска поддерживает вставку PEM-блока или загрузку
файла `.csr` / `.pem` / `.req` / `.txt`. DER не поддерживается;
преобразуйте заранее командой `openssl req -inform DER -outform PEM`.

## Расширения сертификата

Поля, расширяющие базовую модель X.509. Самые важные:

* **Key Usage** — какие операции допустимы для ключа.
* **Extended Key Usage** — назначения «прикладного» уровня (TLS
  server, TLS client, code signing и т. д.).
* **Subject Alternative Name** — DNS-имена, IP-адреса и пр.
* **CDP / AIA / SIA** — точки распространения CRL, OCSP-респондера
  и публикаций УЦ. Подробно — [Точки
  распространения](distribution_points.md).
* **Certificate Policies** — политики выпуска и применения,
  обозначаемые OID-ами.

## CRL и OCSP

* **CRL** — подписанный УЦ список отозванных сертификатов,
  публикуется периодически.
* **OCSP** — протокол онлайн-проверки статуса по конкретному
  сертификату.

## Профиль сертификата

В OwnCA — шаблон, фиксирующий набор расширений (Key Usage, EKU,
политики, OID-поля, переопределения точек распространения). Профили
**глобальные** — они не привязаны к конкретному УЦ. Форма выпуска
просит выбрать одновременно издающий УЦ и профиль; CDP / AIA / OCSP
/ SIA / freshestCRL разрешаются по правилу «профиль перекрывает УЦ».
См. [Профили сертификатов](cert_profiles.md).

## Роли

OwnCA работает с единственной учётной записью — **superuser**.
Разделения на роли нет; все действия (создание УЦ, заполнение
subject и расширений, выпуск, отзыв) выполняются из-под одного
аккаунта.

Очереди заявок и стадии одобрения нет: сертификат создаётся в
момент отправки формы.

## Жизненный цикл

```
[Форма выпуска] → [Выпуск] → [Активный сертификат]
                                    ↓
              [Истечение] | [Отзыв] | [Продление]
```
