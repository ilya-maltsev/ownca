# Выпуск произвольного сертификата (Custom Cert Issue)

`/custom-cert-issue/` — немедленный выпуск сертификата **без очереди
заявок**. Выберите УЦ и опциональный профиль, задайте нужные
расширения и нажмите **Issue certificate**. Сертификат создаётся
сразу же, и вы перенаправляетесь на страницу его деталей.

Страница всегда доступна из пункта меню **Cert Issue** — никакого
переключателя или feature-флага, скрывающего её, нет. Все действия
выполняются от имени единственного суперпользователя.

## Когда использовать

* Тестовые/служебные сертификаты с нестандартным набором расширений,
  для которых неудобно заводить отдельный профиль.
* Подпись сертификата с новым OID, ещё не добавленным ни в один
  профиль.
* Эксперименты — проверить реакцию приложения на определённую
  комбинацию `keyUsage` + критическое `policyConstraints`.

## Поля формы

| Группа | Что задаёт |
|---|---|
| **Issuer CA** | УЦ, который подпишет сертификат (обязателен) |
| **Profile** | опционально — выбор профиля подгружает его KU/EKU/расширения и показывает поля для его custom OID. Пусто = «Free-form (no profile)». |
| **Ключ и срок действия** | алгоритм (`gost2012_256`, RSA, ECDSA, …); для ГОСТ-алгоритмов появляется селектор paramset; срок действия в днях. В списке остаются только алгоритмы, совместимые с выбранным УЦ — см. раздел *Совместимость семейств ключей УЦ и сертификата* ниже |
| **Subject** | CN (обязателен), C, ST, L, O, OU |
| **Subject Alternative Names** | DNS, IP, e-mail, URI, otherName (по строке для многозначных полей) |
| **Key Usage** | чекбокс на каждый бит + флаг критичности (используется в free-form; игнорируется, если KU задан профилем) |
| **Extended Key Usage** | произвольная строка (`serverAuth, clientAuth, …`) + флаг критичности (то же правило) |
| **Точки распространения** | `crl_url`, `aia_url`, `ocsp_url`, `sia_url`, `freshest_crl_url` — пустое значение наследуется от УЦ |
| **Идентификаторы ключей** | переключатели `subjectKeyIdentifier`, `authorityKeyIdentifier`, `AKI include issuer:always` |
| **Custom OID values** | строки *(OID, ASN.1 type, value)* в free-form режиме; обязательные поля — для OID, заданных профилем |
| **Extra extension lines** | произвольный текст, добавляется к расширениям сертификата дословно |
| **External CSR** | опциональная загрузка PEM CSR (или вставка текста) — если задан, сервер подпишет именно его, новый ключ не создаётся |

### Совместимость семейств ключей УЦ и сертификата

УЦ может подписывать сертификаты, **ключ которых принадлежит тому же
криптографическому семейству**, что и собственный ключ УЦ. Семейства:

| Семейство | Алгоритмы |
|---|---|
| `gost` | `gost2012_256`, `gost2012_512` |
| `rsa` | `rsa:2048`, `rsa:4096` |
| `ec` | `ec:P-256`, `ec:P-384` |
| `ed25519` | `ed25519` |

При выборе УЦ в форме список **Algorithm** автоматически
фильтруется по семейству этого УЦ — несовместимые варианты
скрываются, ранее выбранное значение заменяется на первый
допустимый. Сервер выполняет ту же проверку при отправке формы:
запрос с несовместимым `key_alg` отклоняется с сообщением *«Key
algorithm … is not compatible with the issuing CA»*, сертификат
не создаётся.

### Как профиль и free-form объединяются

Если профиль выбран, его расширения берутся за основу. Строки
free-form-формы добавляются только тогда, когда атрибут в начале
строки (`keyUsage`, `extendedKeyUsage`, …) ещё **не** встречается в
строках профиля. То есть профиль, задающий `keyUsage`, не будет
расширен чекбоксами формы; но строки профиля и формы, относящиеся к
разным атрибутам, объединяются.

Поля, определяемые на стороне УЦ (CDP / AIA / OCSP / SIA /
freshestCRL, issuerAltName), подчиняются правилу приоритета:
значение профиля побеждает, значение УЦ используется как fallback,
пустое значение → расширение пропускается.

## Что происходит при отправке

1. Параметры формы объединяются с выбранным профилем (если есть) в
   итоговый набор расширений.
2. Проверяется заполнение обязательных custom-OID полей профиля.
3. Сертификат подписывается издающим УЦ.
4. При успехе — перенаправление на страницу деталей сертификата,
   откуда можно скачать сертификат, ключ, CSR, PEM-бандл или
   PKCS#12-архив.
