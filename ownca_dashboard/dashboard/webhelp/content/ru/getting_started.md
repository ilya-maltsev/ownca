# Вход и первое подключение

Этот раздел описывает первый запуск интерфейса OwnCA, авторизацию и
ориентацию по структуре приложения.

## Предварительные требования

* Развёрнутый экземпляр OwnCA (см. README проекта и `dev_env/`).
* Современный браузер с поддержкой TLS — для GOST-шифрсьютов
  рекомендуется использовать поставляемый nginx-фронтенд из
  `dev_env/nginx/`, собранный с поддержкой
  [`gost-engine`](https://github.com/gost-engine/engine).

## Первый вход

1. Откройте URL развёртывания (по умолчанию `https://localhost:8443`).
2. На странице `/login/` введите имя пользователя и пароль.
3. После успешной аутентификации вы попадёте на обзорную панель `/`.

> Учётная запись администратора по умолчанию — `admin` / `admin` —
> создаётся автоматически. **Смените учётные данные ещё до первого
> запуска**: отредактируйте `DASHBOARD_ADMIN_USER` /
> `DASHBOARD_ADMIN_PASSWORD` в `dev_env/.env` (или в среде) и
> перезапустите контейнер. Формы смены пароля внутри приложения
> нет — значения переменных окружения применяются при каждом
> старте. Подробнее — в разделе [Учётная запись
> администратора](users.md).

## Структура интерфейса

| Зона | Назначение |
|---|---|
| Левая боковая панель | Навигация, переключатель языка, кнопка Help, выход. |
| Верхняя плашка | Заголовок текущей страницы и название проекта. |
| Основная область | Содержимое раздела: таблицы, формы, детальные карточки. |
| Ссылка `Help` в боковой панели | Открывает контекстную справку в новой вкладке. |

## Переключение языка

Переключатель `RU / EN` расположен в боковой панели между блоком
навигации и ссылкой Help. Выбор языка применяется ко всему
интерфейсу и страницам справки.
