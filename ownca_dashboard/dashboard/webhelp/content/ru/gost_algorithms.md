# ГОСТ-алгоритмы

OwnCA поддерживает выпуск сертификатов с подписью по российским
стандартам криптографии.

## Стандарты

| Стандарт | Назначение | Хеш |
|---|---|---|
| **ГОСТ Р 34.10-2012** | Электронная подпись на эллиптических кривых. | — |
| **ГОСТ Р 34.11-2012** («Стрибог») | Хеш-функция длиной 256 или 512 бит. | используется в подписи |
| **ГОСТ 28147-89** / **Кузнечик (Р 34.12-2015)** | Симметричное шифрование. | — |

В сертификатах X.509 OwnCA использует:

| Параметр | 256 бит | 512 бит |
|---|---|---|
| OID алгоритма ключа | `1.2.643.7.1.1.1.1` | `1.2.643.7.1.1.1.2` |
| OID алгоритма подписи | `1.2.643.7.1.1.3.2` | `1.2.643.7.1.1.3.3` |
| Хеш | ГОСТ Р 34.11-2012 / 256 | ГОСТ Р 34.11-2012 / 512 |
| Размер открытого ключа | 512 бит (две координаты по 256) | 1024 бит |

## Параметры эллиптических кривых

Для `gost2012_256` OwnCA по умолчанию использует **paramSetA**
(`1.2.643.7.1.2.1.1.1`). Для `gost2012_512` — **paramSetA**
(`1.2.643.7.1.2.1.2.1`).

Списки рекомендованных наборов параметров — в [ТК 26](https://tc26.ru/).

## Совместимость

* OpenSSL 1.1+, 3.x — через gost-engine.
* CryptoPro CSP 4.0+ — совместим по сертификатам и подписи.
* nginx с патчем gost-tls — поддерживает TLS-сьюты
  `GOSTR341112-256-CNT-MAC` и `KUZNYECHIK-CTR-OMAC`.

## Использование в OwnCA

* На странице [Управление УЦ](cas.md) — выбор алгоритма ключа УЦ.
* На странице [Выдача сертификата](custom_cert_issue.md) — алгоритм
  ключа определяется CSR-ом или, при серверной генерации, выбирается
  из алгоритма по умолчанию.
* Переменная окружения `OWNCA_DEFAULT_KEY_ALG` задаёт алгоритм по
  умолчанию (см. [Конфигурация](configuration.md)).
