# Обслуживание

`/system/maintenance/` — счётчики, статус крипто-бэкенда и
операционные действия.

## Счётчики

Ряд карточек показывает сырые значения, взятые напрямую из БД:

* **Authorities** — общее количество УЦ.
* **Active certs** — сертификаты со статусом `active`.
* **Revoked certs** — сертификаты со статусом `revoked`.
* **Expired certs** — сертификаты со статусом `expired`.
* **CRL exports** — суммарное количество выпусков CRL по всем УЦ.

## Крипто-бэкенд

Панель, показывающая, исправен ли криптостек OwnCA:

* **GOST engine loaded** — зелёный индикатор, когда команда
  `openssl engine gost` отвечает, иначе красный. При красном статусе
  будут работать только RSA / ECDSA / Ed25519 — пока движок снова не
  станет доступен.
* **Storage directory** — путь из `OWNCA_STORAGE_DIR`.
* **openssl version** — полный вывод `openssl version -a` (флаги
  сборки, платформа, путь поиска движков).

## Операции

Панель **Operations** объединяет действия по обслуживанию; каждая
кнопка выводит результат рядом с собой.

### Refresh metadata

Перечитывает каждый файл сертификата с диска и переписывает поля в БД
(subject, даты, fingerprint, серийник). Также все активные
сертификаты, у которых `notAfter` уже в прошлом, переводятся в статус
`expired`.

Применяется после ручных правок файлов в каталоге хранения. Файлы на
диске **не** изменяются — только читаются.

### Rebuild all CRLs

Заново формирует список отзыва сертификатов для каждого включённого
УЦ и записывает новую выгрузку CRL. CRL каждого УЦ перезаписывается с
увеличенным номером CRL и актуальным набором отозванных серийников.

Каждый перестроенный CRL дополнительно публикуется в общий каталог
выгрузки `OWNCA_STORAGE_DIR/crls/` под именем `<имя_УЦ>.crl`
(например, `rsa_root.crl`) — так все CRL можно раздавать из одного
предсказуемого каталога. Основная копия каждого УЦ по-прежнему
хранится в `cas/<uuid>/crl.pem`.

Применяется после массового отзыва или для обновления интервала
`nextUpdate` до запуска планового задания обновления CRL.
