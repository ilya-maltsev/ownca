# Отзыв и CRL

## Отзыв сертификата

Кнопка **Revoke** на странице деталей сертификата помечает сертификат
как отозванный и немедленно формирует новый CRL для издающего УЦ.

### Коды причины

| Причина | Когда используется |
|---|---|
| `unspecified` | Причина не указана. |
| `keyCompromise` | Закрытый ключ скомпрометирован. |
| `CACompromise` | Скомпрометирован ключ издающего УЦ. |
| `affiliationChanged` | Изменилась организация владельца. |
| `superseded` | Заменён новым сертификатом. |
| `cessationOfOperation` | Прекращена эксплуатация. |
| `certificateHold` | Временная приостановка — может быть снята позднее. |

Причина фиксируется в записи CRL. Клиенты, обрабатывающие коды причины,
могут вывести пользователю более информативное сообщение об ошибке.

### Что происходит при отзыве

1. Сертификат помечается отозванным с выбранным кодом причины в
   индексе УЦ.
2. Подписывается новый CRL и заменяет предыдущий для этого УЦ.
3. Журнал выпусков CRL фиксирует номер CRL и количество отозванных
   сертификатов.
4. Статус сертификата переключается на `revoked`, фиксируется
   время отзыва.

## Жизненный цикл CRL

### Генерация CRL по требованию

На странице УЦ нажмите **Regenerate CRL**, чтобы выпустить новый
CRL без отзыва сертификатов. Это нужно, когда срок текущего CRL
подходит к концу.

### Скачивание CRL

**Download CRL** на странице УЦ отдаёт последний CRL в формате PEM.
Если CRL ещё ни разу не выпускался, он будет создан в момент
первого скачивания.
